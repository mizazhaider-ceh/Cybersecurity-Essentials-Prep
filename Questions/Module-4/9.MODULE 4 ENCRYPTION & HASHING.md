## ğŸ¯ KEY EXAM TIPS FOR MODULE 4

### **âš¡ CRITICAL CONCEPTS:**

1. **Encoding vs Encryption** [MOST IMPORTANT!]
    - Encoding: NO KEY, reversible, well-known algorithm
    - Encryption: NEEDS KEY for security
2. **Symmetric vs Asymmetric**
    - Symmetric: 1 key, fast, bulk encryption, key distribution problem
    - Asymmetric: 2 keys, slow, short messages, solves key distribution
3. **Hashing Properties**
    - Fixed length output
    - One-way (can't reverse)
    - Collision-resistant
    - Used for digital signatures
4. **Digital Signature = Integrity + Authentication + Non-repudiation**
5. **PKI Components**
    - CA: Signs certificates
    - RA: Verifies users
    - Certificate: Public key + info + CA signature

### **ğŸ“ KEY ALGORITHMS TO KNOW:**

- RC4 (stream, weak)
- DES (56-bit, weak)
- 3DES (DES 3x)
- AES (128/192/256-bit, current standard)

**Asymmetric:**

- RSA (1024-4096 bit)
- ECC (smaller keys, less power)

**Hashing:**

- MD5 (128-bit)
- SHA-1 (160-bit)
- SHA-256 (256-bit)
- SHA-512 (512-bit)
