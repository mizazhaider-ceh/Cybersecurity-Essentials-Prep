**Question 157** (üî¥ **CRITICAL - PENTEST vs VULN SCAN!**)

**Differences between Penetration Tests and Vulnerability Scans:**


| Aspect | Vulnerability Scan | Penetration Test |
| :-- | :-- | :-- |
| **Goal** | Identifying, ranking, reporting vulnerabilities | Identifying ways to EXPLOIT vulnerabilities to bypass security controls |
| **When** | At least quarterly or after significant changes | At least annually and after significant changes |
| **How** | Automated tools + manual verification | Mostly MANUAL + may use automated tools |
| **Reports** | Potential risks of known vulnerabilities (NVD/CVSS scores) | Description of each vulnerability + specific exploitation methods + specific risks |
| **Duration** | Short (seconds to minutes per host) | Days or weeks (increases as more assets discovered) |

**Match:** "Automated scan finding known CVEs" = ?

**Answer:** Vulnerability Scan

**Match:** "Security expert manually attempting to escalate privileges" = ?

**Answer:** Penetration Test

### **üìù CRITICAL FORMULAS \& CONCEPTS:**

**Risk Formula:**

- **Risk = Likelihood √ó Impact**
- **Likelihood = f(Threat, Vulnerability, Controls)**
- **Inherent Risk** = risk WITHOUT controls
- **Residual Risk** = risk AFTER controls

**Risk Responses:**

1. **Reduction/Mitigation** (implement controls)
2. **Avoidance** (don't do risky activity)
3. **Transfer** (insurance, outsource)
4. **Acceptance** (live with it)

**Risk Assessment Inputs:**

1. Asset identification
2. Threat assessment
3. Vulnerability assessment

**Vulnerability Types:**

- Technical
- Process-related
- Organizational
- Emerging

**Pentest Phases:**

1. Planning
2. Discovery (OSINT, scanning, banner grabbing, social engineering)
3. Attack (exploit, escalate, install tools)
4. Reporting (with mitigation recommendations!)

### **üîß KEY DISTINCTIONS:**

**Vulnerability Scan vs Penetration Test:**


| Feature | Vuln Scan | Pentest |
| :-- | :-- | :-- |
| Method | Automated | Manual |
| Goal | Find vulnerabilities | Exploit vulnerabilities |
| Frequency | Quarterly | Annually |
| Duration | Minutes | Days/weeks |

**Pentest Types:**

- **Black box** = no knowledge
- **White box** = full knowledge
- **Grey box** = partial knowledge
- **External** = outside perimeter
- **Internal** = inside perimeter
- **Covert** = teams not informed

**Risk Approaches:**

- **Ad hoc** = random, no strategy (BAD!)
- **Compliance-based** = checklist mentality
- **Risk-based** = identify specific risks (BEST!)
