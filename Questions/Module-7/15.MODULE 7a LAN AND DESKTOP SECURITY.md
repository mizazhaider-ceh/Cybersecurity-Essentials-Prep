**Question 5** (üî¥ **CRITICAL - LAN COMPONENTS!**)

**Repeater:**

- **OSI Layer:** Physical layer
- **Function:** Amplifies and repeats signal

**Hub:**

- **OSI Layer:** Physical layer
- **Function:** Amplifies and repeats signal to number of other physical lines

**Bridge:**

- **OSI Layer:** Data link layer
- **Function:** Connect 2+ LANs into one LAN, can filter; usually connects wireless and wired networks

**Switches:**

- **OSI Layer:** Data link \& Network layer
- **Types:**
    - Layer-2 switches (based on MAC address)
    - Layer-3 switches (based on IP address)
- **Note:** Usually NO WAN port to connect to Internet; used between parts of LAN

**Router:**

- **OSI Layer:** Network layer
- **Function:** Examines packets and "routes" them onward or blocks them; often includes firewall
- **Note:** Usually HAS WAN port to connect to Internet

**Match:** "Device operating at Data Link layer that forwards traffic based on MAC address" = ?

**Answer:** Layer-2 Switch

**Question 59** (üî¥ **CRITICAL - WEP!**)

**802.11: WEP (Wired Equivalent Privacy):**

- **Encryption:** Symmetric using **RC4** (considered **VERY WEAK ‚âà broken**)
- **Implementation:** Weak implementation of encryption protocol
- **Key management:** ONE key for all data, not changed frequently (key distribution problem)
- **Security status:** Passwords can be **cracked in MINUTES** on standard laptop!

**True or False:** "WEP is considered secure for modern use."

**Answer:** False (WEP is BROKEN!)

**Question 60** (üî¥ **CRITICAL - WPA/WPA2/WPA3!**)

**802.11: Wi-Fi Protected Access (WPA, WPA2 \& WPA3):**

**WPA improvements over WEP:**

- **One key PER PACKET** (instead of one key for all data)
- Based on known WPA code (password or "passphrase")
- **Stronger integrity check:**
    - Prevent replay (e.g., same bank transaction can't be performed twice)
    - Prevent modifications

**WPA2:**

- **AES is MANDATORY** (from WPA2 onwards)

**WPA3:**

- Measures for limiting **weak WPA codes** (weak passwords/passphrases)

**Match security level (weakest to strongest):**

**Answer:** WEP < WPA < WPA2 < WPA3

- User does **NOT have free access** to hardware

- In **isolation** of one another
- Example: VMware

**Creates layer between hardware and guest OSs:**

- Manages shared processing and memory resources on host
- Often called **HYPERVISOR**

**What is the management layer called?**

**Answer:** Hypervisor

**Question 193** (üî¥ **CRITICAL - VIRTUALIZATION PROBLEMS!**)

**Virtualization - Potential Security Problems:**

1. **Inadequate host configuration** ‚Üí vulnerabilities affecting host AND guests
2. **Exploits of host vulnerabilities** or **DoS attack** ‚Üí affects ALL guests
3. **Compromise of management console** (hypervisor console) ‚Üí unapproved admin access to all guests
4. **Insecure protocols for remote access** ‚Üí exposure of administrative credentials
5. **Data leak between guests** if memory not released/allocated by host in controlled manner

**Most critical:**

**Answer:** \#3 - Management console compromise = full control!

**Question 211** (üü° Medium - **DEFINITIONS!**)

**Operational Technology (OT):**

- All systems used to manage **industrial operations** (vs IT managing administrative operations)
- Examples: production line management, mining operations control, oil \& gas monitoring

**Industrial Control Systems (ICS):**

- Systems used to **monitor and control industrial processes**
- Example: power consumption on electricity grids

**SCADA (Supervisory Control And Data Acquisition):**

- Display process under control
- Provide access to control functions

**What does SCADA stand for?**

**Answer:** Supervisory Control And Data Acquisition

***

**Question 212** (üî¥ **CRITICAL - INDUSTRIAL CHARACTERISTICS!**)

**Industrial Systems Characteristics:**

- **Historically:** Designed as stand-alone systems
- **NOT designed with security in mind**
- Often have OT **proprietary protocols** (e.g., **Modbus**)
- Must work in **real-time** ‚Üí often no 'overhead' software (e.g., security checks)
- **Nowadays:** More often networked
- Strong evolution towards network devices for better operational control/reporting
- **AIC instead of CIA** (Availability, Integrity, Confidentiality)
- **Long lifespan** compared to IT devices

**What priority model do industrial systems use?**

**Answer:** AIC (Availability first!) instead of CIA

***

**Question 213** (üî¥ **CRITICAL - INDUSTRIAL IMPACT!**)

**Industrial Systems - Impact of Attack:**

Could be **SEVERE** (e.g., gas tank monitoring equipment):

- **Catastrophic failure** ‚Üí business paralyzed
- Possibly **LIFE-THREATENING**

**Possible Mitigating Controls:**

- Firewall
- **Network segmentation** (minimally OT/IT separation; also isolate vulnerable systems like Windows XP)
- Authentication
- **Change default passwords**
- Encryption (where possible)
- Updates

**Why is industrial system security critical?**

**Answer:** Attacks can be life-threatening and cause catastrophic failures
